# Cohort ITE Comparison — Program-Level Analysis

Compare multiple residents' completed analyses to surface program-level gaps vs. individual gaps.

**Prerequisite:** Each resident must have a completed `analysis_v2.json` (generated by the analyze-ite command).

Reports directory:

If arguments provided via $ARGUMENTS, use those as resident directory paths.

Otherwise, find all available analysis files:
```cmd

```

List what's found and ask the user: "I found analyses for [names]. Which residents should be included in this cohort comparison?"

Each `analysis_v2.json` must have an `analysis_version` key — skip any files missing this key (they're from the old v1 pipeline).

## Step 2: Read report_config.json

Read:
```
C:\Users\mpsch\Desktop\board_prep_intel\03_module.3_analyst\scripts\report_config.json
```

Note `weakness_threshold` — this defines "weak" consistently across all residents.

## Step 3: Load All Analysis JSONs

Read each selected `analysis_v2.json`. For each resident, extract:
- Name
- Exam year
- PGY level
- Overall % and scaled score
- Blueprint performance by category (% correct per category)
- Body system performance by system (% correct per system)
- Weak areas list (already computed by pipeline)
- Easy misses list

## Step 4: Cross-Resident Analysis

### 4a. Score Overview Table
Build a table: resident × {overall %, scaled score, pass tier, percentile}.

### 4b. Blueprint Gap Classification
For each blueprint category:
- Count how many residents score below `weakness_threshold`
- **Program gap**: >50% of residents below threshold → curriculum issue
- **Individual gap**: only 1 resident below threshold → individual study target
- **Universal strength**: all residents above threshold

### 4c. Body System Heatmap
Same classification logic applied to body systems. Build a cross-tab table: residents × systems, color-coded by performance band:
- 🟢 ≥80%
- 🟡 70–79%
- 🔴 <70%

### 4d. Shared vs. Individual Weakness Summary
Produce two lists:
1. **Program-level gaps** (majority weak) — recommend curriculum intervention
2. **Individual gaps** (1 resident weak) — recommend individual study plan

### 4e. Easy Miss Overlap
Check if multiple residents share the same easy misses (same QIDs). These are the highest-priority review items — if 3+ residents missed an "easy" question, it's a curriculum miss.

## Step 5: Generate Cohort Report DOCX

Use the docx skill to generate a styled report. Apply St. Luke's palette (navy/blue/gray), Aptos font, US Letter.

Save as:
```
C:\Users\mpsch\Desktop\board_prep_intel\03_module.3_analyst\reports\ITE_{year}_Cohort_Comparison.docx
```

Report sections:
1. **Cohort Overview** — score table, pass rates
2. **Blueprint Analysis** — program gaps vs. individual gaps
3. **Body System Heatmap** — cross-tab table
4. **Shared Weakness Priority List** — ranked by how many residents are weak
5. **Individual Weakness Summary** — per-resident, bullet-pointed
6. **Easy Miss Overlap** — shared easy miss QIDs with question text
7. **Program Recommendations** — 3–5 concrete curriculum actions
8. **Individual Recommendations** — per-resident study focus (1–2 items each)

## Step 6: Present Results

Present the DOCX file link and summarize the top 3 program-level findings in plain language:
- Which categories are program-wide gaps
- Which residents need individualized attention
- The single highest-yield curriculum action
