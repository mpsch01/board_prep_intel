# ITE Pipeline — Failure Mode Catalog

Known failure modes, root causes, and fixes. Check here before asking the user to re-run anything.

---

## 1. `ModuleNotFoundError: No module named 'fitz'`

**Trigger:** Running `ite_analyze_v2.py` fresh install.
**Root cause:** PyMuPDF not installed.
**Fix:**
```cmd
pip install pymupdf
```
If `pip` is not found:
```cmd
"C:\Users\mpsch\AppData\Local\Programs\Python\Python312\python.exe" -m pip install pymupdf
```

## 2. DOCX Not Generated — `npm` / `docx` Module Missing

**Trigger:** JSON output appears but DOCX file is missing.
**Root cause:** `ite_report_builder_v2.js` requires the `docx` npm package. One-time install per machine.
**Fix (run once in the scripts directory):**
```cmd
cd "C:\Users\mpsch\Desktop\board_prep_intel\03_module.3_analyst\scripts"
npm install docx
```

## 3. `'python' is not recognized` / `'py' is not recognized`

**Trigger:** Running the CLI command in Windows Terminal or Command Prompt.
**Root cause:** Python is not on PATH for this shell session.
**Fix:** Always use the full Python path:
```
C:\Users\mpsch\AppData\Local\Programs\Python\Python312\python.exe
```
Never use `python`, `py`, or `python3` in commands given to Mikey. Always expand to the full path.

## 4. Empty Weak Areas — Analysis Completes But No Practice Questions

**Trigger:** Analysis runs successfully but `weak_areas` list is empty; 0 practice questions generated.
**Root cause (A):** Resident scored above `weakness_threshold` in all categories → no weak areas identified. Genuinely strong performance.
**Root cause (B):** Category string mismatch — parsed PDF categories don't match DB strings. See `blueprint_categories.md` for valid strings.
**Diagnosis:**
- Check the `analysis_v2.json` for the `scores_by_blueprint` and `scores_by_body_system` arrays
- If all values are above threshold → Cause A (real result)
- If any values are null or the category names look unusual → Cause B (parsing failure)
**Fix for Cause B:** Verify the PDF parser output against `blueprint_categories.md`. Look for the 2024 body system renaming issue if processing a 2024 exam.

---

## 5. Unicode / Encoding Error in Resident Name

**Trigger:** Script errors on output path when resident has special characters in their name (accents, apostrophes).
**Root cause:** Windows file system + Python path handling conflict on non-ASCII characters.
**Fix:** Use an ASCII-safe version of the name in `--output-dir`. E.g., `O'Brien` → `OBrien`. The DOCX title can still use the proper name — only the folder needs to be ASCII.

---

## 6. PDF Parse Returns Zero Rows

**Trigger:** Parser runs but extracted data is completely empty.
**Root cause (A):** Wrong PDF provided — user handed blueprint PDF as body system or vice versa.
**Root cause (B):** PDF is a scanned image, not machine-readable text. ABFM provides text PDFs, but a scanned printout won't work.
**Root cause (C):** PDF is a different ABFM report type (e.g., Overall Score Report vs. Item Performance Report).
**Fix:** Confirm the PDF contains "Item Performance Report By Blueprint Category" or "Item Performance Report By Body System Category" in the header. If the PDF header says something else, it's the wrong document type.

## 7. `report_config.json` Not Found

**Trigger:** Script errors with "config file not found" or similar.
**Root cause:** The config file was not created yet or the path is wrong.
**Fix:** Confirm `report_config.json` exists at:
```
C:\Users\mpsch\Desktop\board_prep_intel\03_module.3_analyst\scripts\report_config.json
```
If missing, recreate from the spec in `references/report_config_spec.md`. The skeleton with defaults takes 2 minutes.

---

## 8. `analysis_v2.json` Missing `analysis_version` Key

**Trigger:** `cohort-compare` command skips a resident's file.
**Root cause:** The file was generated by the old v1 pipeline (before the rebuild).
**Fix:** Re-run `analyze-ite` for that resident to produce a fresh v2 output. The old file can stay in place — the new run will overwrite it.

---

## 9. ICD-10 Crosswalk Returns No Links

**Trigger:** `icd10_crosswalk` section shows 0 articles linked.
**Root cause:** `question_icd10` table may not have coverage for the specific weak-area questions, OR the `--plugins icd10` flag was not passed.
**Diagnosis:**
- Check CLI command — confirm `--plugins "concept,icd10"` is present
- Check `analysis_v2.json` for `icd10_plugin_active: true`
- Run: `SELECT COUNT(*) FROM question_icd10` — should be 5,218. If 0, the table is empty.

---

## 10. SSL Certificate Error (VA / Government Sites)

**Context:** Seen during PDF download steps, not the main analysis pipeline. Documented here for completeness.
**Root cause:** DoD Certificate Authority not trusted by Python's certificate bundle.
**Fix for download scripts:**
```python
import requests, urllib3
urllib3.disable_warnings()
response = requests.get(url, verify=False)
```
This is appropriate for internal/government sites with known cert issues.

## General Diagnostic Checklist

When a run fails and the error isn't obvious:

1. **Check the full Python path** — `python` / `py` / `python3` all fail; use full path.
2. **Check the working directory** — some scripts require being run from `scripts/` not project root.
3. **Check `npm install`** — DOCX builder requires it once per machine.
4. **Check category strings** — compare parser output vs. `blueprint_categories.md`.
5. **Check `report_config.json` exists** — file is required at runtime.
6. **Paste the full traceback** — last 10 lines usually pinpoint the issue.
