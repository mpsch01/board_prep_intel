    V4 Ideas

    Where Module 3 Could Go

    1
    Longitudinal Resident Record
    A per-resident JSON profile that accumulates each checkpoint's analysis. Run it in July, save it. Run it in October — the output shows delta: what improved, what's new, what's still stuck. This is the infrastructure everything else depends on.
    Implementation: resident_profiles/ folder + profile schema (resident_id, checkpoints[], improvement_vectors). Small addition, massive value.
    Low effort · Highest value
    2
    One-Page Action Summary
    Alongside the full DOCX, generate a clean single-page output: overall score, 3 priority weak areas, 5 must-read articles, 10 practice questions. Something a resident can tape to a wall and actually use. No jargon, no 20-page document.
    Implementation: new template in ite_report_builder — same data, different slice. Or a standalone Python script that reads analysis_v2.json and outputs a 1-page PDF/DOCX.
    Low effort · High value
    3
    Clinical Pathway Output
    Surface the Layer 3 data properly. For each top weak area, show the clinical pathway: what to screen for → how to diagnose → first-line treatment → second-line → monitoring. Articles organized by clinical role, not just citation count. Completely different reading experience.
    Data is all there — 4,020 clinical_pathways rows. Just needs a new output section in the report builder.
    Low effort · High value
    4
    Video Course Bridge
    Build a mapping table: body system + blueprint category → VC session numbers. When a resident is weak in cardiovascular, the analysis says "Sessions 3, 7, 12 cover this — watch these first." The direct connection between score gaps and the structured curriculum that doesn't exist yet.
    Requires: vc_session_map table (48 rows: session_number, title, body_system[], blueprint_category[]). Could be built manually from the video course outline — one-time data entry.
    Medium effort · High value
    5
    Cohort HTML Dashboard
    Replace the static cohort DOCX with an interactive self-contained HTML file. Filter by body system, sort by resident, see year-over-year trends across the class. Faculty can open it in any browser — no software needed. Color-coded heatmap of the whole cohort at a glance.
    Implementation: data:interactive-dashboard-builder skill + existing analysis_v2.json files as source. Could be generated automatically after each cohort run.
    Medium effort · High value
