# ABFM Board Prep — File Naming Convention Spec
**Owner:** Mike, MD
**Established:** 2026-03-12
**Applies to:** `00_canonical/` outputs and all named working files across `claude_knowledge/`

---

## Core Logic

The filename alone should tell you:
1. What kind of file it is (prefix)
2. What it contains (descriptor)
3. Whether it's the active canonical version (leading `00_`)
4. What time period it covers (year suffix, when relevant)
5. Whether it's an older version (version suffix on archived files only)

---

## Prefix Taxonomy

| Prefix | Stands For | File Types | Use For |
|---|---|---|---|
| `EX_` | Exam | docx, pdf, gdoc | Exam-facing documents — qbanks, study guides, question sets |
| `RPT_` | Report | docx, pdf | Analysis reports, written summaries, findings documents |
| `AVC_` | AAFP Video Course | docx, xlsx, pdf | Course integration files — session maps, outlines, crosswalks |
| `REF_` | Reference | docx, xlsx, pdf | Literature reference files — tier lists, acquisition lists |
| `DB_` | Database | xlsx, csv, .db | Structured data files — any tabular or database format |

---

## The `00_` Active Indicator

A leading `00_` marks the **active canonical deliverable** — the file that is current, complete, and ready to use or share.

**Rules:**
- `00_` prefix applies to canonical output files only
- Only ONE file per type should have the `00_` at any time
- When a file is superseded, remove `00_` and add `_vn` before archiving
- `00_` files sort to the top of the folder automatically in Windows Explorer — intentional
- `00_` was chosen over `^` because `^` is not a valid Windows filename character

**Active:** `00_EX_qbank_20-25.xlsx`
**Archived predecessor:** `EX_qbank_20-25_v1.xlsx` → moved to `previous_versions/`

### Dual-Format Rule

When the same dataset exists in both `.xlsx` and `.csv` format:
- `00_` applies to the `.xlsx` only — it is the human-facing, shareable version
- The `.csv` gets no `00_` — it is the pipeline/script-facing export
- Both share the same descriptor and year suffix

**Example:**
```
00_DB_qbank_master_20-25.xlsx   <- primary deliverable, human-facing
DB_qbank_master_20-25.csv       <- script-facing export, no 00_
```

## Year Suffix

- **Date range:** `_yy-yy` (e.g. `_20-25` for 2020–2025)
- **Single year:** `_yy` (e.g. `_26` for 2026)
- **Omit** when the file is not time-bound (e.g. a reference guide, a naming spec)

---

## Version Suffix

- Applied to **archived/superseded files only** — never to the active file
- Format: `_v1`, `_v2`, `_v3` etc.
- Active file has no version tag — the absence of `_vn` means current

---

## Full Naming Pattern

```
[00_]PREFIX_descriptor[_yy-yy][_vn].ext
```

| Token | Required | Notes |
|---|---|---|
| `00_` | Only on active canonical files | Leading sort prefix, no space after |
| `PREFIX_` | Always | One of the five prefixes above |
| `descriptor` | Always | Snake_case, concise, no redundant words |
| `_yy-yy` | When time-bound | Year range or single year |
| `_vn` | Archived files only | Never on active file |
| `.ext` | Always | Lowercase — **do not append manually. Windows manages the extension. The `.ext` shown in this document is for reference only.** |

## What This System Does NOT Cover

- **Scripts** (`.py`, `.js`, `.bat`) — names left as-is, managed by folder location
- **JSON files** — names left as-is, managed by pipeline conventions
- **BATON.md / README files** — fixed names by convention, not subject to this system
- **Intermediate working files** — inside pipeline subfolders, not subject to this system

---

## When to Apply This System

- **New files generated by the pipeline** — apply at creation time
- **Existing files** — apply during reorganization sweep (see RENAMING_PROPOSAL.md)
- **Archived files** — add `_vn` suffix before moving to `previous_versions/`
- **Shared/exported copies** — use the canonical name as-is, no modification needed

---

## Applying to New File Types

If a new file type arises that doesn't fit the existing prefixes, add a row to the prefix taxonomy table above before naming the file. Do not improvise prefixes mid-project.

## BATON File Management

### Archive Rule
Before a new BATON is written, the current BATON must be:
1. Copied to `house_keeping_hub\baton_archive\[project_name]\`
2. Renamed with project tag + datestamp: `BATON_[project]_YYYYMMDD.md`
3. Then the live `BATON.md` is overwritten with the new session handoff

### Folder Structure
```
house_keeping_hub\
  BATON.md                          <- always the current/live handoff
  baton_archive\
    atlas_v2_reorg\                 <- one folder per project
      BATON_atlas_v2_reorg_20260312.md
    ite_intelligence\
      BATON_ite_intelligence_20260311.md
    [project_name]\
      BATON_[project]_YYYYMMDD.md
```

| ITE Intelligence Pipeline | `ite_intelligence` |
| Guideline Extractor | `guideline_extractor` |
