Master Key — Node Types · Script Reference · File Lifecycle

Answers: (1) what enters each module first  ·  (2) what each script does and why  ·  (3) outputs and destinations  ·  (4) what happens to each file after it lands

Node Color Legend

Primary Source — raw input not generated by any script (exam PDFs, AAFP website, ABFM score reports).
Python Script — dark navy. Active production logic.
Node.js Script — dark green. Document rendering or synthesis.
Batch / Shell Orchestrator — dark gray. Windows .bat files that chain multiple scripts.
Excel / Manual Curation — foundational source for DB build. Human-curated.
Active Pipeline File — output consumed by the next script. Light green.
Database / DB Table — source of truth or enrichment tables. Light purple.
DOCX / Terminal Deliverable — final output, not further processed. Light blue.
Log / Audit File — human-review artifact. Not processed by scripts. Light yellow.

File Lifecycle Categories

Every file belongs to exactly one lifecycle category:
Functional Script
.py or .js executing pipeline logic. Kept permanently in scripts/. Updated when behavior changes.
Database / Source of Truth
ite_intelligence.db and its tables. Never deleted. Backed up before any rebuild.
Active Pipeline File
JSON or PDF moving between stages. Consumed by the next script. Disposable with care — regenerable.
Terminal Deliverable
DOCX or HTML final output. No further script processing. Delivered to user or resident.
Audit Log
JSON/TXT written for human review or traceability. Never read by scripts. Archive after review.
Archive / Backup
DB backups, v1 scripts, test folders. Preserved for rollback. Not in active pipeline.
