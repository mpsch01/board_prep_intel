# Security Policy

## Supported Versions
This is a research/educational project; support is best-effort on the main branch.

## Reporting a Vulnerability
Please report vulnerabilities via GitHub Security Advisories if enabled. If not, you can report them by:
- Emailing Mikey directly.
- Opening a minimal public issue without including sensitive details.

## Information to Include in a Report
When reporting a vulnerability, please provide the following information:
- Description of the vulnerability
- Steps to reproduce
- Impact assessment

## Expected Response Timeline
- Acknowledgment within 72 hours
- Initial assessment within 7 days

## Guidelines for Responsible Disclosure
We appreciate your help in keeping our project secure. Please do not disclose any vulnerabilities to the public until we have had the opportunity to address them.

## Note About Secrets
Do not include API keys or Protected Health Information (PHI) in your reports. Please rotate any keys that may have been exposed.

## Scope
This repository contains medical education content and does not operate any live patient systems.

## Safe Harbor
We welcome good-faith research and testing of our security measures. We will not take legal action against anyone who reports a vulnerability responsibly, according to this policy.
