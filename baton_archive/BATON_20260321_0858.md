## KEY ARCHITECTURE REMINDERS

- **4-module architecture is the new north star:** Warehouse (M1), Processor (M2), Analyst (M3), Sandbox (M4). Do not revert to old module numbering (⓪–Ⓔ+Ⓕ).
- **AAFP course is a lens/priority filter, not a module.** It deposited the VC gate into the Warehouse and is now static context.
- **VC gate (session_hy_inserts_v7.json) is the sole criterion** for $right_click$ vs local_lite tier assignment. DB membership alone is not sufficient.
- **Codon filename format:** `Author_Year#@#ART-XXXX@#@.pdf` — start codon `#@#`, stop codon `@#@`.
- **5 locked design principles still apply:** (1) VC outline is primary gate, (2) ART-ID must carry tier info, (3) Fix data not code, (4) No files moved until migration plan written and tested, (5) local_lite is a genuine product.
- **Source data is protected, derived data is disposable.** DB + PDFs + VC gate survive everything. DOCXs + enriched JSONs + crosswalk indexes can be regenerated.
- **The user thinks in data lifecycle:** acquisition → normalization → storage → enrichment → presentation. Each module should serve one phase.
