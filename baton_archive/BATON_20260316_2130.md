## What Was Done This Session

### FLAG 13 Layer 3 — Clinical Pathways / Blending Engine (COMPLETE)

**Core insight:** The ABFM exam subcategories already encode article classification — no Claude API calls needed.

Built `build_clinical_pathways.py` — three-phase zero-cost engine:

**Phase 1 — Classify:** 1,366 articles classified into 5 engine types using subcategory + source_type + categories:
- Subcategories (Screening→preventive, Diagnosis→diagnostic, Treatment→acute, etc.)
- Pharmacology/Management disambiguated by article categories (chronic vs acute)
- RCT source types (NEJM, JAMA, Lancet, BMJ) override subcategory
- 31 articles unclassifiable (non-cited stubs with no linked questions)

**Phase 2 — Build:** ENGINE_ROLE_MAP routes (engine_type × icd10_relevance) → pathway_role:

| Role | What it means |
|---|---|
| screening_prevention | USPSTF-style criteria, risk tools, prevention protocols |
| diagnosis | Diagnostic criteria, "which test" logic, staging systems |
| first_line | Initial treatment — the always-appropriate early step |
| second_line | Novel agents, RCT alternatives, escalation |
| monitoring | Ongoing management, lab intervals, red flags |
| referral | When to refer, specialist thresholds |
| special_pops | Pediatric, geriatric, pregnant, renal-adjusted dosing |

**Phase 3 — Report:** 4 CSVs + sample pathway JSON.

**Result: 4,528 clinical_pathways rows, 7 pathway roles, $0 API cost.**

**Implication:** All 216 PDFs should be re-extracted via v2.3 pipeline for content-rich DOCXs. This is a batch operation (~$0.03/file × 216 = ~$6.50) that can run locally.

## Key Architecture Reminders

- **Codon format:** `Author_Year#@#ART-XXXX@#@.pdf` — `#@#` = start, `@#@` = stop
- **Pipeline order:** convert_pdfs_to_json.py → ite_intelligence_enricher_batch.py → build_crosswalk_index.py
- **v2.3 pipeline:** main.py → synthesize.js → ite_intelligence_enricher.py → build_summary.js (3 Claude API calls)
- **Baton lifecycle:** archive → sync docs → write new (always in this order)
- **Baton naming:** active = `BATON_active_YYYYMMDD_HHMM.md` at root; archived = `BATON_YYYYMMDD_HHMM.md` in baton_archive/
- **Enriched JSON folder:** `03_enriched_JSON` (underscore, no space)
- **Enrichment v4:** codon_filename → high confidence; clean_ref → high confidence; no other strategies
- **Trend queries:** Always use `body_system_merged` (not `body_system`) for longitudinal analysis — 2024 taxonomy change
- **Join paths:** `question_ref_pairs` (canonical, uses clean_ref) or `qid_art_xref` (ergonomic, uses article_id)
- **ICD-10 join chain:** `icd10_rollup` → `icd10_code_xref` → `article_icd10` → `articles` → `qid_art_xref` → `questions`
- **Pathway join chain:** `clinical_pathways` → `articles` → `qid_art_xref` → `questions`
- **Engine classification:** subcategory → engine_type (zero cost, from ABFM exam data)
- **Standard filters:** Exclude `citation_count = 0`, `source_type = 'stub'`, `article_id = 'ART-0001'`
