### DB-Guided Extraction Built (v1.0)
New approach replacing the old 3-API-call pipeline:

**Core insight:** The pre-computed DB intelligence (concept_tags with diagnoses/drugs/guidelines/thresholds, question stems, explanations, exam trends) acts as an extraction guide. Claude makes one focused call per file instead of three blind ones.

**New scripts created:**

| Script | Purpose |
|---|---|
| `db_guided_extractor.py` | Core engine: DB clue assembly + single Claude call → structured extraction + synthesis |
| `batch_db_extract.py` | Batch API wrapper (submit/poll/write) for 50% cost savings |
| `batch_reprocess.ps1` | Rewritten PowerShell orchestrator with 5 modes |

**Prompt design:** Each file receives a clue package containing:
- Article metadata (categories, tier, source_type, citation_count)
- Hot-zone diagnoses/drugs/thresholds (frequency-ranked across all linked questions)
- Full question details (stems, correct answers, explanations, concept_tags)
- Exam trend direction (increasing/stable/decreasing)
- Raw pdfplumber text

Claude returns the standard extraction+synthesis schema that `build_summary.js` consumes.
