## What Was Done This Session

### Path C — DB-Only DOCX Library (COMPLETE)

**Goal:** Generate one DOCX study guide per classified article using ONLY data from `ite_intelligence.db` — zero API calls, zero cost.

**Process:**

1. **Template co-authoring** — Used iterative feedback loop to design a study-guide DOCX template. Started from the v1 visual DNA in `build_summary.js` (colors, fonts, spacing), then layered in DB intelligence sections. Multiple revision rounds on section order, formatting, alignment, font choice, and page-break behavior.

2. **Exemplar (ART-0370)** — Built `build_exemplar_v2.js` for ADA Standards of Care 2023. Hardcoded data to validate exact visual output. Four revision rounds:
   - v1: Initial template → user preferred v1 formatting style over new design
   - v2: Rebuilt with v1 visual DNA + reordered sections per user feedback
   - v3: Fixed alignment, added colored concept text, switched to Aptos font, removed header
   - v4 (final): Added `cantSplit: true` on all table rows to prevent page breaks mid-table

3. **Batch generator** — Built `build_db_docx.js` from finalized template. Uses `sql.js` (pure JS SQLite) + `docx` (npm docx-js). Queries all classified articles, pulls ICD-10 codes, pathway roles, linked questions with concept_tags, trend data from CSVs. Generates template-driven study notes via `generateStudyNote()`. Dynamically colorizes concept text from `concept_tags` JSON.

4. **Filename collision fix** — Initial run produced 1,295 files for 1,366 articles (71 collisions from shared canonical names like "American_2013"). Patched filename format to `<canonical>_<ART-ID>_db_intel.docx`. Re-ran. 1,366 files, zero collisions.

5. **Verification** — Spot-checked via pandoc text extraction. All sections rendering correctly: pathway maps, ITE intelligence tables, colored concept columns, correct answers, study notes. File sizes range 9.2KB–15KB.

### Finalized DOCX Template Sections (in order)

1. **Title Banner** — Article title, author/year/source, full citation
2. **Classification Badge** — ITE Citations count + Body Systems (2-column table, amber background)
3. **Clinical Pathway Map** — ICD-10 codes with relevance level and pathway role (4-column table)
4. **ITE Exam Intelligence** — Linked questions with year, QID, question stem (truncated), concept tested (with colored text: green=diagnoses, yellow=drugs, red=thresholds)
5. **Correct Answers** — Answer key with body system and subcategory context
6. **Key Testable Concepts** — Unique concepts extracted from concept_tags, color-coded
7. **Exam Trend Context** — Body system trend data (slope direction, exam year coverage)
8. **Study Note** — Template-driven hybrid-tone narrative built from DB data
9. **High-Yield Concepts** — Summary of most important testable items
10. **Footer** — Article ID + codon/canonical filename

### Visual DNA

- **Font:** Aptos (all text)
- **Colors:** NAVY `1F3864`, BLUE `2E75B6`, LBLUE `D6E4F7`, DGRAY `595959`, TEAL `E2EFDA`, DTEAL `2D572C`, AMBER `FFF2CC`
- **Concept chips:** Green `00B050` (diagnoses), Yellow `FFD700` (drugs), Red `FF6B6B` (thresholds), White `CCCCCC` (other)
- **Page:** US Letter, 0.75" margins (1080 DXA)
- **Tables:** All rows have `cantSplit: true` — no mid-table page breaks

## Key Architecture Reminders

- **Codon format:** `Author_Year#@#ART-XXXX@#@.pdf` — `#@#` = start, `@#@` = stop
- **DB DOCX filename format:** `<canonical>_<ART-ID>_db_intel.docx` (collision-proof)
- **DB DOCX dependencies:** `npm install docx sql.js` in the `docx_build/` directory
- **DB DOCX usage:** `node build_db_docx.js [--limit N] [--article ART-XXXX]`
- **Pipeline order:** convert_pdfs_to_json.py → ite_intelligence_enricher_batch.py → build_crosswalk_index.py
- **v2.3 pipeline:** main.py → synthesize.js → ite_intelligence_enricher.py → build_summary.js (3 Claude API calls)
- **Path C pipeline:** ite_intelligence.db → build_db_docx.js → DOCX (0 API calls)
- **Baton lifecycle:** archive → sync docs → write new (always in this order)
- **Baton naming:** active = `BATON_active_YYYYMMDD_HHMM.md` at root; archived = `BATON_YYYYMMDD_HHMM.md` in baton_archive/
- **Enriched JSON folder:** `03_enriched_JSON` (underscore, no space)
- **Enrichment v4:** codon_filename → high confidence; clean_ref → high confidence; no other strategies
- **Trend queries:** Always use `body_system_merged` (not `body_system`) for longitudinal analysis — 2024 taxonomy change
- **Join paths:** `question_ref_pairs` (canonical, uses clean_ref) or `qid_art_xref` (ergonomic, uses article_id)
- **ICD-10 join chain:** `icd10_rollup` → `icd10_code_xref` → `article_icd10` → `articles` → `qid_art_xref` → `questions`
- **Pathway join chain:** `clinical_pathways` → `articles` → `qid_art_xref` → `questions`
- **Engine classification:** subcategory → engine_type (zero cost, from ABFM exam data)
- **Standard filters:** Exclude `citation_count = 0`, `source_type = 'stub'`, `article_id = 'ART-0001'`
