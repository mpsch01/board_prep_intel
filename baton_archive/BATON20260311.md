## WHAT WAS BUILT THIS SESSION

### match_tiers_to_library.py — Parser Fix (2 bugs)
File: board_prep/ite_refs/05_scripts/match_tiers_to_library.py

**Bug 1 — Title parser (period-format refs):**
  Before: Only extracted title words from colon-format refs (e.g. "Smith DK: Title")
          Period-format refs (e.g. "Smith DK. Title. Journal.") fell through to a
          fallback that grabbed page numbers instead of title words.
          Affected: 129 refs. Fixed: 115/125 (10 edge cases are org-committee refs
          with no parseable author block — genuine hard cases, not bugs).
  After:  Detects format via AUTHOR_COLON_RE. Colon-format → takes text after author
          colon up to journal name. Period-format → skips author block at first ". "
          then extracts title up to journal name via JOURNAL_RE.

**Bug 2 — Author parser (single-author colon refs):**
  Before: search_zone was clean_ref.split(':')[0] — stripped the colon, so
          single-author refs like "Ramirez J:" had no delimiter after the initial
          and the regex returned empty.
  After:  For colon-format refs, search_zone includes the colon: clean_ref[:first_colon+1]
          Added ':' to the delimiter set in the author regex.
          Fixed: Ramirez, Saguil, Chamberlain and all other single-author AFP refs.

Net improvement across 3 runs:
  Run 1 (baseline):       163 matched
  Run 2 (title fix):      192 matched  (+29)
  Run 3 (author fix):     197 matched  (+5, Must-Read 15→17)

## KEY ARCHITECTURE REMINDERS

**Codon filename format:** LastName1_LastName2_Year#@#ART-XXXX@#@.pdf
**Pipeline flow (extract_guideline.bat):**
  [1/5] Extract (ingestion.py — QID parse built in)
  [2/5] Synthesize (synthesize.js)
  [3/5] ITE Enrichment (ite_intelligence_enricher.py --file)
  [4/5] Build DOCX (build_summary.js)
  [5/5] Done
  [6/6] Calibrate (if --calibrate flag or first run)

**ite_intelligence block written to:** top-level data.ite_intelligence
  (build_summary.js checks both data.ite_intelligence and extraction.ite_intelligence)
**citation_display written to:** source.citation_display

**linked_qids structure:**
  { qid, exam_year, question_stem, concept_tested (plain text),
    concept_colors: [{text, color}] sorted green→yellow→red }

**TF-IDF buckets:** top tertile = green, middle = yellow, bottom = red
  Green = concept specific to this article's exam history
  Red = concept that appears broadly across the question bank

**Enricher prompt limits (CURRENT):**
  _build_payload: full stem and explanation stored (no truncation)
  Prompt sends: stem[:800], keywords[:150], explanation[:1000]

**match_tiers_to_library.py thresholds:**
  CONFIDENT_THRESHOLD = 0.60  (MATCHED)
  LOW_THRESHOLD       = 0.35  (POSSIBLE)
