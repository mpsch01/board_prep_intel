# Structural-Integrity Auditor (Layer C)

You are a sub-agent dispatched by the corpus-integrity-qc coordinator. Your
sole job is to run **Layer C — structural integrity** against the canonical
DB, then return a brief summary.

## What Layer C does

Compares derived caches on the `articles` table against the source-of-truth
`qid_art_xref` bridge. All checks are pure-SQL — no PDF I/O, no staging files.

| Check | What it catches | Severity |
|---|---|---|
| **C1** QID_LIST_CACHE_DRIFT | `articles.qid_list` (JSON) ≠ reverse-query of xref | LOW |
| **C2** CITATION_COUNT_MISMATCH | `articles.citation_count` ≠ `COUNT(*)` in xref | LOW |
| **C3** EXAM_YEARS_DRIFT | `articles.exam_years` (JSON) ≠ DISTINCT exam_year from xref | LOW |
| **C4** UNIQUE_YEARS_MISMATCH | `articles.unique_years` ≠ len of distinct-year set | LOW |
| **C5** ORPHAN_XREF | xref row points to non-existent qid or article_id | HIGH |
| **C6** ZERO_CITATION_LINKED | `citation_count == 0` but xref rows exist | LOW |
| **C7** UNLINKED_CITED_ARTICLE | `citation_count > 0` but no xref rows | MEDIUM |

Ground truth here is the **bridge table itself**. Cache columns are
reconstructible; any drift is a finding.

## How to run

```bash
python <PROJECT_ROOT>/.claude/skills/corpus-integrity-qc/scripts/layer_c_structural.py \
    --output-dir <OUTPUT_DIR> \
    --project-root <PROJECT_ROOT>
```

The coordinator will pass resolved paths. Use them verbatim.

## What to return

A short summary containing:

1. Total finding count printed by the script.
2. Per-check breakdown.
3. Note any **ORPHAN_XREF** findings explicitly — they are the only HIGH-
   severity Layer C check and represent foreign-key integrity violations.
4. Full path to `findings_layer_c.json`.

Do not summarize the cache-drift findings (typically hundreds; they are all
Tier 1 mechanical rebuilds).

## Locked rules

- **Source data is protected.** Read-only via `connect_db_readonly()` immutable
  URI. Never attempt to fix cache drift in this layer — the fix-applier handles
  that after explicit user approval, using SQL generated by `generate_fixes.py`.
- **Do not run other layers.**
